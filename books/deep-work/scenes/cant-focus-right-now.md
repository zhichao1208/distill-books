# Scene: can't focus right now

## Trigger

User in present tense: "I can't focus" / "I keep scrolling" / "I'm so distracted right now" / "I've been on TikTok for an hour."

Distinct from `week_review` ("I had a bad week") and `structural` ("my schedule is bad") — this one is about the *current moment*, not the system.

## After main SKILL.md cold-start A, enter this scene

### 1. The immediate intervention beats the schedule

Newport's framing: a single deep session done right beats 5 days of half-attempts. Don't try to redesign the week before the user finishes today.

```
We're not solving your week right now. We're rescuing the next 30 minutes.
Everything else is later.
```

### 2. The 30-minute defended block

```
Right now, do these in order:

  1. Phone: face down, in a drawer or on a different floor (3-meter rule)
  2. Browser: close every tab except one. Yes, all of them.
  3. Slack / Teams / WhatsApp / iMessage: closed, not minimized
  4. Door: closed if you have one. Headphones: on if you don't.

This isn't "focus tips." This is *removing the infrastructure of distraction*.
You can't out-willpower a phone within reach.
```

### 3. Name the one thing (smallest scope possible)

The user's natural answer to "what are you working on" is too big. Cut it.

> User: "I'm working on the Q3 report."
> Skill: "30 minutes isn't a report. What's the one section you need to draft right now? Title only is fine."

> User: "I'm trying to write code for the new feature."
> Skill: "30 minutes isn't a feature. What's the one function or one diagram? Name it specifically."

> User: "I need to think about my career."
> Skill: "30 minutes isn't 'thinking about my career.' What's the one question? Write down the question."

The rule: the deliverable must be **nameable in one sentence** and **plausible in 30 minutes**.

### 4. Set the timer (literal — not metaphorical)

```
Set a timer for 30 minutes. Right now. Not your phone — the watch on
your wrist, the kitchen timer, the clock on the wall. If you must use
the phone, set the timer first then put the phone away.

When it rings, the block is done. You don't get to re-up "just 10
more minutes." That's how the next 5 hours go.
```

### 5. The attention-residue rule

Newport's most actionable single insight from the book:

```
When the timer rings: stand up. Walk away. 10 minutes minimum before
the next thing.

What just happened in your brain is real work. The residue from a
30-min deep block takes 10-20 minutes to clear, whether you give it
the time or not. Skipping the gap doesn't save time — it bleeds the
next thing.
```

### 6. Closer

```
30 minutes. One named thing. Timer set. Phone away.
Go.

Tomorrow (or in 2 hours), come back if you want. Say "did it" /
"didn't" / "started but lost it at minute 12" — any of those is useful.
"Lost it at minute 12" tells us the cue or the scope was wrong, and we
re-pick.

Not coming back is also fine.
```

---

## Out of scope for this scene

- **User wants to redesign the whole week** → run cold-start C, go to philosophies. This scene is for *now*, not for *every week*.
- **User wants a productivity system (GTD, PARA, Notion templates)** → those are systems, not Deep Work's frame. Acknowledge and redirect to the philosophy question.
- **User is in chronic distraction (every day this for months)** → not this scene; run vulnerable_clinical or burnout state check.

---

## Failure cases

- User does the 30 minutes, comes back saying "I didn't actually focus, I just stared at the screen."
  → That's data. The likely cause: residue from earlier today wasn't drained. Suggest a 5-minute reset (walk, water, no screens) before the next block.
  
- User says "I can't put my phone in another room because I need it for work."
  → Validate. The fallback: phone face down, *across* the room, with notifications silenced. Worse than another room, better than at-hand.
  → If even this is impossible (on-call, parent monitoring), the user's job/life is in the `structural` or `structural_shallow` bucket. Re-route.

- User does 30 minutes, comes back saying "I did 3 hours of deep work, what's next."
  → Don't add more. Newport's data: most knowledge workers max out at 2-4 hours of deep work per day, *total*. More than that is either fake-deep, or unsustainable. Tomorrow's block should still be 30-90 min. Add reps, not duration, week over week.
