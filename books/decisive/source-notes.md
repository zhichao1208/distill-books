# Decisive · sources

## Book
- *Decisive*, Chip Heath & Dan Heath, Crown Business, 2013
- ISBN: 978-0307956392 · Chinese: 《决断力》, 中信出版社, 2013
- Heath brothers' books: Made to Stick (2007), Switch (2010), Decisive (2013), The Power of Moments (2017), Upstream (2020)
- Goodreads ~4.06 / ~30k ratings

## Core methodology — WRAP framework
- **W — Widen Your Options** — counter narrow framing
  - Vanishing options test
  - Multitracking parallel options
  - Opportunity cost prompt
  - "What would I tell a friend"
- **R — Reality-Test Your Assumptions** — counter confirmation bias
  - Consider the opposite
  - Disconfirming questions
  - "Ooching" — small experiments before commit
- **A — Attain Distance Before Deciding** — counter short-term emotion
  - 10/10/10 rule
  - "What would my successor do"
  - "What would I advise a friend"
  - Honor core priorities
- **P — Prepare to Be Wrong** — counter overconfidence
  - Premortem (Gary Klein's technique, cited)
  - Bookending (worst-case + best-case)
  - Tripwires (pre-committed reassessment points)
  - Set aside acceptable losses

## Critical honesty
- **Storytelling style** — case studies are post-hoc rationalized; Heath brothers write in popular-business style.
- **Doesn't address forced-choice / no-good-options situations** — WRAP can't manufacture an option that doesn't exist.
- **Doesn't address information-vacuum** — WRAP adds process to ignorance; more steps don't fix missing data.
- **Concept ownership** — much of the framework synthesizes existing work (Kahneman / Klein / Welch / Kross). Heaths are curators; intellectually honest skill should attribute.

## Differentiation from apple-ouyang/book-to-skill's existing Decisive skill
That existing skill uses a router architecture (sub-skills for each WRAP step, neutral primitives). Our skill:
- Single SKILL.md format with boot + every-reply rules
- Opinionated about WHEN WRAP is overkill (low-stakes / reversible) or underkill (forced choice / values question)
- End-to-end consultation scenes, not primitive sub-skills
- Named failure modes inline

Both skills can coexist; users may prefer either based on style preference.
