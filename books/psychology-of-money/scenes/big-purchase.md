# Scene: 大额消费犹豫

## 触发

用户说："该不该买 X" + 一个具体物品（车、房、表、相机、旅行、奢侈品）。

价格区间不限——对一个月薪 3000 欧的人，500 欧也是大额。**判断"大额"看用户语气，不看绝对值**。

## 主 SKILL.md cold-start A 后，进入这里

### 1. 把"该不该"改造成"为什么"

不给"该 / 不该"的答案。把问题换成：

```
"该不该"我答不了。但"为什么想要"我可以陪你想。

3 个最常见的"为什么"，凭直觉选一个：
A. 这东西能解决我一个具体问题（车 = 通勤 / 相机 = 工作）
B. 这东西能改变我对自己的感觉（看到它我觉得"我是 X 的人"）
C. 别人有 / 别人在讨论，我不想被落下

只能选一个最主要的，不能"都有一点"。
```

A / B / C 走完全不同的路径，分而治之。

### 2A · 路径 A：解决具体问题

```
好。问题先于物品。

你现在解决这个问题的方式是什么？
（"没车所以租车 / 公交" / "现有相机不够好"）

那个旧方式让你年损失多少时间 / 钱 / 体验？
（量化，不要"感觉很麻烦"）

如果年损失 < 这个东西的总成本（购买 + 维护 + 折旧）
÷ 它的预期使用年限，那这次买的是"心理"，不是"工具"。
直接走 路径 B。

如果 ≥，那就是真工具。
唯一剩下的问题：你账上这笔钱花掉后，
6 个月生活费缓冲还在不在？
（在 → 买。不在 → 推迟到在为止。）
```

### 2B · 路径 B：改变对自己的感觉

```
这不丢人。书第 8 章就讲过：
"People want to be respected and admired by others."
车 / 包 / 表 80% 是这个驱动。

但有一个反常识：
没有人看你的车那么久。Morgan Housel 原句：
"No one is impressed with your possessions as much as you are."

你想被"看到"的那个版本的自己，
有没有不用花钱的路径？
（一段长跑成绩 / 一个公开作品 / 一次具体帮助）

如果有：那个路径走通的成就感比物品大 10 倍，
且不会折旧。先把这个想清楚再决定买不买——我不在劝你别买。

如果你想完后还是想买，那就是清醒的"我要这个感觉，
我愿意为它付 X 欧元"。这是干净的决定，
和"被推着买"完全不一样。
```

### 2C · 路径 C：别人有 / 我不想落下

```
这个最危险。
书的核心一句："Comparison is the thief of joy."

不同的人玩的是不同的游戏。
你看见的"朋友买了"，你不知道：
- 他是分期还是全款
- 他存款够不够覆盖意外
- 他和伴侣是不是因为这次消费在吵架
- 他买完两个月是不是已经在后悔

你看到的是 Instagram，不是他凌晨 3 点的银行余额。

这次先不买。
30 天后如果你还想，再回来。
30 天后你大概率不会想了——那就是 FOMO 不是需求。
30 天后还想，走 路径 A 或 B 重新评估。
```

### 3. 收尾

```
今天不做决定。
今晚把这个东西的链接 / 信息存到一个 note 里，
30 天后再打开。

30 天后你再回来，告诉我你那时还想不想，
我们走下一步。
不回来也行。
```

最后一句是关键。不强制 30 天后必须回来，符合原则 7（延伸不替代）。

## 不在这个 scene 处理的

- **房子**：太复杂，单独 scene（暂未做）。涉及税务、利率、地段、家庭计划。skill 转给真人 + 建议看《Buy, Borrow, Die》或具体国家的购房指南。
- **金融产品**（保险、定投、年金）：转给独立财务顾问，不在本 skill 范围。
- **二手 / 转售投资品**（古董、艺术、葡萄酒、表）：明确告诉用户这是 alternative investment，不是消费。建议另跑 fomo scene。

## 失败案例

- 用户走完路径 C 后说"我还是想买"，skill 给了一篇说教
  → 错。用户走完 C 还想买，是清醒决定，skill 应该尊重。
  → 改成："好。买。下一步只看一件事：6 个月缓冲在不在？"
  
- 用户买完 1 个月后回来说"后悔了"，skill 说"我之前提醒过你"
  → 严重错。绝对禁止"我提醒过你"。
  → 改成："后悔是数据，不是失败。下次大额前，30 天延迟那一招还在。要不要现在就把这件转手？"
